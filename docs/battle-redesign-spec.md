# 戰鬥界面改版規格 v1

## 改版目標

戰鬥頁是「記帳輸入」與「怪物回饋」的合體。玩家進入這頁的主要目的仍是快速記帳；怪物、血量、攻擊特效與掉落回饋負責讓記帳有遊戲感。

## 核心流程

1. 選擇分類。
2. 輸入金額與備註。
3. 送出記帳攻擊。
4. 怪物扣血並播放目前裝備的攻擊特效。
5. 今日紀錄更新。
6. 若怪物被擊敗，顯示掉落預告、任務進度與結算徽章入口。

## 版面分區

### 1. 今日討伐頂列

- 返回、頁名、日期。
- 今日剩餘預算只作為小型狀態，不重複大卡片。

### 2. 怪物舞台

- 怪物名稱、等級語氣、HP 條。
- 怪物本體要是視覺主角。
- 怪物腳下保留舞台陰影或魔法圈，不能浮在背景裡。
- 第一階段可用 CSS 疊層做戰鬥語氣：魔法陣、封印線、暖色聚光與低對比暗角。
- 戰鬥頁不得長得像主頁套裝展示；即使暫用同一張背景，也要用舞台層與 UI 讓它讀作討伐場。
- 攻擊特效在舞台中央播放，未來由商店收藏的攻擊特效替換。

### 3. 今日紀錄

- 今日紀錄在戰鬥頁只顯示薄型摘要。
- 可看最近幾筆與筆數，不和記帳面板搶主視覺。
- 修改與刪除仍可用，但不是第一視覺。

### 4. 記帳攻擊面板

- 分類、備註、金額、加總計算、記帳攻擊。
- 預覽本筆攻擊力，但不讓傷害數字壓過金額輸入。
- CTA 只有一個主要按鈕：記帳攻擊。

### 5. 擊敗回饋

怪物擊敗後顯示小結算，不只顯示完成文字。

- 淨化成功。
- 掉落一般券預告。
- 任務進度提示。
- 結算徽章提示。

實際發獎仍遵守每日結算與既有邏輯；戰鬥頁先顯示預告，不提前發正式結算獎勵。

## 與商店連動

- 攻擊特效商品顯示位置：戰鬥頁記帳攻擊動畫。
- 結算徽章顯示位置：擊敗回饋、每日結算、地圖戰報。
- 任務刷新券不在戰鬥頁使用，仍放任務頁每日任務區。

## 第一階段實作

1. 調整戰鬥頁資訊層級。
2. 今日紀錄變成薄型摘要。
3. 擊敗回饋加入掉落與徽章預告。
4. 保留現有怪物與背景，不先重畫資產。

## 後續項目

- 重畫或統一怪物美術：保持可愛財務壓力感，不做恐怖怪。
- 製作專屬戰鬥背景：帳本廣場、任務告示板、封印魔法陣、可站立地面與遠景學院。
- 接上商店攻擊特效裝備。
- 接上結算徽章正式獲取與展示。
- 加入多人公會共同怪物或月度 Boss。
