# 窮鬼勇者 Visual System v1

## 美術核心

**日系魔法學院 × 溫暖財務冒險。** 畫面要讓記帳像一次輕巧的冒險行動，但財務數字永遠比裝飾更清楚。

### 三個視覺關鍵字

- 溫暖：奶油紙張、日光金、低刺激陰影
- 魔法：星術紫只負責品牌與稀有內容
- 清楚：深靛文字、薄荷安全色、珊瑚警示色各司其職

## 色彩規則

| 角色 | Token | 色值 | 用途 |
|---|---|---:|---|
| 品牌主色 | `--color-brand-500` | `#7161B8` | 目前頁籤、主要品牌標記 |
| 品牌深色 | `--color-brand-700` | `#433A78` | 深色標題、焦點邊框 |
| 獎勵金 | `--color-reward-500` | `#E7AD36` | 黃星、獎勵、限定亮點 |
| 安全薄荷 | `--color-safe-500` | `#2FAF9F` | 預算安全、完成狀態 |
| 警示珊瑚 | `--color-danger-500` | `#D96573` | 超支、失敗、破壞性操作 |
| 主文字 | `--color-ink-900` | `#27304A` | 主要數字與文字 |
| 次文字 | `--color-ink-500` | `#77738D` | 說明文字 |
| 奶油底 | `--color-surface-base` | `#FFF8EC` | App 底色 |

規則：一個元件最多一個強調色；粉紅不再作為通用裝飾色，只用於警示或特定收藏品。

## UI 形狀與材質

- 面板圓角：24px；小元件：14px；膠囊：999px。
- 面板使用暖白半透明材質，不使用純白冷玻璃。
- 一般面板只用一層陰影；重要 CTA 可以增加金色內光。
- 每頁只能有一個主 CTA。
- 背景負責世界觀，UI 負責資訊；UI 不再和背景競爭細節。

## 圖示系統

- 導航／功能圖示：圓潤 2D、2.5px 等效描邊、單色為主。
- 貨幣／獎勵物件：2.5D 道具，可有高光與材質。
- 禁止混用 emoji 作正式圖示。
- 同組輸出統一 64×64 畫布及 8px 安全邊距。
- 狀態只使用 `default / active / disabled` 三種；active 由底板與顏色共同表達。

## 完整主題套裝收藏系統

每份獎勵是一套完整角色演出，不拆成可自由混搭的紙娃娃零件。收藏價值來自新鮮感，而不是只替換上衣顏色。

### 每套必須獨特的八個面向

1. 髮型與髮飾：長度、輪廓、綁法或頭飾不得沿用同一樣板。
2. 臉部造型：保留主角辨識度，但改變妝感、眼神、表情與主題細節。
3. 服裝輪廓：不只換色，剪裁、重心與材質都要有差異。
4. 隨身物：一至二件有故事性的物件，必須由角色穿戴或手持，不得漂浮在旁邊。
5. 專屬背景：每套都有獨立場景，光源、色溫與套裝一致。
6. 場景擺飾：只留二至三個必要視覺線索，不做商品展示架式堆疊。
7. 場景呼應：角色手持物、視線、身體朝向、腳底位置需與背景元素形成關係；不能只是把角色貼在背景前。
8. 專屬動作：動作要與場景互動，不得全部套裝共用呼吸、眨眼或同一轉圈。
9. UI 氣氛：主 CTA、安全色與微光可随主題調整，但不犧牲數字可讀性；避免制式白底玻璃卡，HUD 材質要跟場景同一套語氣。

### 主角一致性邊界

- 必須保留：基礎年齡感、Q 版比例、主色基因與整體人格。
- 必須改變：髮型輪廓、主表情、服裝外輪廓、隨身物與動作語彙。
- 不使用角色後方的白霧展示框；角色應直接站在專屬背景裡。
- 腳底必須有明確場景落點，頭頂不貼 HUD，完整動作不得進入玩家資訊卡。

### 資產契約

- 正式版只保留透明 Animated WebP，原始影片不進入前端專案。
- 角色、服裝、髮型與隨身物合成同一動畫資產，避免動作脫節。
- 每套動畫需檢查正面、側面與動作最大幅度幀。
- 每套獨立設定腳底基準點與顯示尺寸，不共用一組 `top/width`。

### 首頁 HUD 與場景空間

- 預算資訊只在下方行動面板出現一次：`今日消費 / 今日可用預算`。
- 上方 HUD 只放角色身份、稱號、成長入口與收藏貨幣，不重複預算進度條。
- 身份與貨幣可收斂為同一個場景式狀態板；材質跟隨套裝，如夏天用清透水光、夜櫻用深色燈籠光。
- 下方行動面板至少要有三個有意義的狀態：今日消費、今日攻擊力、今日可用預算；避免左右資訊之間留下像缺資料的空白。
- 角色腳下不放姓名貨幣大卡，中央區域優先留給角色動作、腳底落點與背景景深。
- 記帳步驟只在新手教學中出現，不常駐於首頁。
